Task 1 · Classical · Digraph
Playfair Cipher
function buildMatrix(keyword): mat = unique chars of keyword + remaining A–Z (skip J) return 5×5 matrix of mat function playfairEncrypt(text, key): mat = buildMatrix(key) pairs = makePairs(text) // handle XX→XQ for each [A, B] in pairs: [r1,c1] = find(mat, A) [r2,c2] = find(mat, B) if r1 == r2: // same row out = mat[r1,(c1+1)%5], mat[r2,(c2+1)%5] elif c1 == c2: // same col out = mat[(r1+1)%5,c1], mat[(r2+1)%5,c2] else: // rectangle out = mat[r1,c2], mat[r2,c1] // Decrypt: same rules, shift left/up instead
Same row: shift right (enc) / left (dec)
Same col: shift down (enc) / up (dec)
Rectangle: swap column indices (both ways)
J treated as I. Repeated letters get X inserted.

Task 6 · Asymmetric · Public Key
RSA — Step by Step
RSA security = difficulty of factoring n = p×q. Key insight: anyone can encrypt with public (e,n) but only you can decrypt with private d. Watch every modular arithmetic step.
Task 6 · Asymmetric · Public Key
RSA Algorithm
// KEY GENERATION choose primes p, q n = p × q // public modulus φ(n) = (p-1)(q-1) // Euler totient choose e: gcd(e, φ(n)) = 1 d = e⁻¹ mod φ(n) // private key public key = (e, n) private key = (d, n) // ENCRYPTION (anyone) function encrypt(M, e, n): return C = M^e mod n // DECRYPTION (private key holder) function decrypt(C, d, n): return M = C^d mod n // Works because: M^(ed) ≡ M (mod n) // Security: hard to factor n = p×q
Encrypt: C = Mᵉ mod n
Decrypt: M = Cᵈ mod n
Key gen: d = e⁻¹ mod φ(n)
Security: Integer Factorization Problem
PropertyValue

Task 7 · Key Exchange · Discrete Logarithm
Diffie-Hellman Key Exchange
Alice and Bob never send the secret. They exchange public values. An eavesdropper sees g, p, A, B — but cannot compute the shared key without solving the discrete logarithm problem.
Task 7 · Key Exchange
Diffie-Hellman
// PUBLIC parameters (shared openly) p = large prime g = generator (primitive root mod p) // Alice: a = random private integer A = g^a mod p // send to Bob // Bob: b = random private integer B = g^b mod p // send to Alice // Shared secret computation: Alice: S = B^a mod p Bob: S = A^b mod p // Both equal g^(ab) mod p ✓ // Eve sees p,g,A,B — cannot find a or b // without solving Discrete Log: a = log_g(A)
Alice public: A = gᵃ mod p
Bob public: B = gᵇ mod p
Shared: S = Bᵃ mod p = Aᵇ mod p = g^(ab) mod p
Security: Discrete Logarithm Problem (DLP)

Task 8 · Asymmetric · Discrete Logarithm
ElGamal Cryptography
Built on DH. Encryption produces two values (C1, C2). C1 hides the session key k. C2 = M × y^k blends the message with the key. Decryption peels off y^k using private key x.
Task 8 · Asymmetric · DLP
ElGamal Cryptosystem
// KEY GENERATION p = large prime, g = generator x = private key (random) y = g^x mod p // public key public = (p, g, y) private = x // ENCRYPTION function encrypt(M, p, g, y): k = random session key C1 = g^k mod p C2 = M × y^k mod p return (C1, C2) // DECRYPTION function decrypt(C1, C2, x, p): s = C1^x mod p // = y^k sInv = modInverse(s, p) M = C2 × sInv mod p return M
Pub key: y = gˣ mod p
Encrypt: C1 = gᵏ mod p, C2 = M·yᵏ mod p
Decrypt: M = C2 · (C1ˣ)⁻¹ mod p
Security: Decisional DH Problem (DDH)
